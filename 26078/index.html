Config in code instead of Config file.

Wiki Link: [discussion:26078]
robertmatthews
Apr 17, 2008 at 1:30 AM
 Does Unity have the ability to have a piece of code, often called a configuration module, do the configuration?

http://code.google.com/p/autofac/wiki/StructuringWithModules

The idea is to have more compile-time checks and refactoring support rather than having to hunt down configuration in hard-to-read configuration files.
ctavares
Apr 17, 2008 at 3:20 AM
 Yes.

You can write a container extension that calls into the container's configuration API's but doesn't actually add any new functionality. This is pretty easy.
