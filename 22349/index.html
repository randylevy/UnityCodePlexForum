Feb 18, 2008 at 3:28 PM
 One of the scenarios I'd like to see covered is the ability for attributes to know what type they were used with.

For example, consider the TraceSourceAttribute used within the CAB to facilitate the injection of a TraceSource instance. While the current approach inverts the control of creating this dependency, the object is still required to specify what the name of the TraceSource will be. What I would consider to be "full" IoC in this case would be that an object would only declare that it needs a TraceSource, but wouldn't care whether it is given the same one every other type uses, one specific to its type, one based upon a logical association within a config file, etc. To do this with the old Object Builder, you would have to register a new Strategy. However, if attributes were passed the type they are being used within then this could be done without adding the additional overhead associated with the use of Strategies.

Derek
